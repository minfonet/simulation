# English-Only Project Artifacts

## Context

Project documentation and memory contained a mix of English and Spanish prose. Mixed-language artifacts increase handoff cost for agents and collaborators, make search less reliable, and create inconsistent expectations for future skills, tests, commits, and PR text.

## Decision

All repository artifacts must be written in English. This includes documentation, memory/session files, code comments, test names/descriptions, PR and commit text generated by agents, and future skills.

Existing non-English content should be translated when touched. Translation must preserve code identifiers, paths, route names, API names, class/interface names, acronyms, role names, and meaning.

## Consequences

- New project artifacts created by agents must use English.
- Documentation and memory updates must avoid introducing non-English prose.
- Historical non-English content should be translated opportunistically when files are edited.
- `docs/06-engineering/language-policy.md` is the source policy document, and `.memory/active-context.md` references it as a mandatory workflow/project rule.
