# Language Policy

## Mandatory rule

All repository artifacts must be written in English.

This applies to documentation, memory/session files, code comments, test names and descriptions, PR and commit text generated by agents, future skills, and any other repository-maintained prose.

## Handling existing non-English content

- Translate existing non-English prose when touching a file.
- Preserve code identifiers, paths, route names, API names, class/interface names, acronyms, role names, and technical meaning.
- Do not translate user-facing product strings solely because they are examples of product behavior unless the task explicitly changes those strings.
- If a task cannot translate all discovered historical content in one pass, document the remaining files and why.

## Agent workflow requirement

Agents must check this policy before creating or editing project artifacts and must keep new Markdown, comments, tests, PR text, commit text, and future skills in English.
